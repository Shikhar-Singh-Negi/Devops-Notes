# Runners

Runners are the processes that pick up and execute CI/CD jobs for GitLab

Jenkins has Agents

# Variables

variable store information that you can use in job scripts.

Types of variables:-

	* Protected :- Only expose to protected branches or protected tags
	* Masked :- Hidden in job logs. Must match masking requirements.(Very Important)
	* Expanded :- variable with $ will be treates as the start of referencr to another variable.

** Note:- We use masked variable to hide/protect our credentials

## Self hosted Runners , SaaS Runners(instance Runners)

1. First we have to install GitLab Runner on instance
2. Link the gitlab runner
3. And add tags on the .gitlab-ci.yml file in jobs in which we want to run the code on different instance


Note:- If the runner not work properly the go to 
		1. /home/gitlab-runner/
		2. ls -a
		3. .bash_logout (comment the data)
		4. gitlab-runner run
