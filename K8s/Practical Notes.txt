## Taints/Tolerations

Taints and tolerations are Kubernetes mechanisms that allow nodes to repel specific pods (taints) while enabling other pods to allow scheduling onto those nodes (tolerations). Taints (applied to nodes) and tolerations (applied to pods) act as a pair to ensure, or restrict, workload placement on specialized nodes.

# To taint
# Example             #name                 # Key-value
#  kubectl taint node tws-cluster-worker2 prod=true:NoSchedule

# To remove taint
#  kubectl taint node tws-cluster-worker2 prod=true:NoSchedule-

# tolerate me hum pod.yml ke ander key-value dal ke run kar sakte hai

## Horizontal Pod Autoscaler (HPA) and Vertical Pod Autoscaler (VPA)

# HPA
  It increase number of replicas of pod 
  It is mainly used in apps/ stateless sets
  HPA (Horizontal): Adds more replicas. Best for stateless apps, web services, and sudden, high-traffic loads.

# VPA
  It increase resource of a pod(100M --> 500M)
  It is mainly used in statefulset.
  VPA (Vertical): Increases CPU/Memory of existing pods. Best for stateful apps, applications with fixed capacity, or to right-size resource requests.

# KEDA (Kubernetes Event-Driven Autoscaler)

  Focus: Scales the number of replicas, but based on external event sources.
  it use HPA to scale based on specific metrics.
  It choose what to use HPA or VPA based on matrics calculations.

* in minikube

## Helm package manager  (Most Important)

first install Helm

It will create all the menifest file of all we need like deployment, ingress, service, etc

We dont have to write any this we just have to provide values in values.yml

** Service ports change karna padega
** aur persenal app ko dalna hai tho values me change karna hai
Steps:-
  1. To create a Chart
    helm create apache-helm

  2. Enter to the apache-helm/

  3. install tree to see how helm created 
      tree

  4. open values.yml
      edit/update values as you want
  
  5. the we only have to package it
      helm package apache-helm/

  6. then install chat 

      ## For dev 
                  # name we want          # name of namespace you want
      helm install dev-apache apache-helm -n dev-apache --create-namespace

      ## For prod
      ## For dev 
                  # name we want          # name of namespace you want
      helm install prod-apache apache-helm -n prod-apache --create-namespace

  7. To uninstall Chart
      helm uninstall dev-apache


## If we want to upgrade Chart

1. go to value.yml upgrade it
    vim apache-helm/values.yml

2. upgrade helm Chart
    vim apache-helm/Chart.yml

3. update file
  helm package apache-helm

4. To upgrade 
  helm upgrade prod-apache ./apache-helm -n prod-apache

5. To rollBack update
                        # revision number
  helm rollback prod-apache 1 -n prod-apache
