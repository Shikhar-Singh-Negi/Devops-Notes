#### Storage

## persistent volumes(PV) & persistent Volume Claim(PVC)

When we creat a pod it runs on a server, it can delete, destroy, etc anytime
so the data inside it can aulo be delete anytime. To prevent this we create volumes
to bind/persist data with host. we requir persistent volume or persistent volume claim.

# We create persistent volume
  ye persistent Volume bannne ke baad ab kisko use karne ke liye claim karna padega
  using persistentVolumeClaim.yml

** Ex:- we first creat pv and then claim in the deploymenty.yml
    Steps:- 1. first we create pv 
            2. and then claim that in pvc and name this as local-pvc
            3. and then use the name local-pvc to claim the storage in deployment

## These storage are in docker container
if you want to go inside the storage container
 #  docker exec -it <Container id> bash

## Taints/Tolerations

Taints and tolerations are Kubernetes mechanisms that allow nodes to repel specific pods (taints) while enabling other pods to allow scheduling onto those nodes (tolerations). Taints (applied to nodes) and tolerations (applied to pods) act as a pair to ensure, or restrict, workload placement on specialized nodes.

# To taint
# Example             #name                 # Key
#  kubectl taint node tws-cluster-worker2 prod=true:NoSchedule

# To remove taint
#  kubectl taint node tws-cluster-worker2 prod=true:NoSchedule-
